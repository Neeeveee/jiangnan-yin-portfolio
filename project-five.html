Project Boundary

Bee Cue is not designed to replace beekeepers' decisions. It provides readable, traceable and comparable information to support judgment under ecological uncertainty.

Design Challenge

Ecological timing is becoming harder to read.

Climate change is disrupting the timing between plant flowering and bee activity. For beekeeping practice, this makes it harder to judge nectar availability, bee activity and environmental risks through experience alone.

01 Climate Change Is Worsening

Fluctuations in temperature, precipitation, and extreme weather are making ecological rhythms even more unstable.

02 Irregular Nectar Flow Patterns

When the flowering season begins earlier, is delayed, or is shortened, it becomes difficult to predict the optimal window for nectar collection.

03 Difficulty in Making Decisions on-Site

The interplay of weather, flowering periods, and the condition of the bee colonies increases the difficulty for beekeepers in making decisions.

04 Fragmented Information

Information from multiple sources lacks a unified structure, making it difficult to directly translate into practical judgments.

Research

Understanding the ecological and beekeeping context

The research connected ecological literature, field context and data structure so the system could be grounded in real beekeeping conditions.

01 Literature Review

The research began with phenological mismatch, pollination, climate change and agricultural decision support systems.

Source: China Meteorological Administration, Climate Change Blue Book
02 Field Context

Field observation focused on the beekeeping site, nectar plant distribution, seasonal changes and local environmental conditions.

03 Data Sources

The system was built around three main types of information: weather data, flowering data and bee activity data.

Key Insight

From scattered signals to practical judgment

The problem is not only a lack of data, but the difficulty of translating scattered ecological signals into practical judgment.

01 Beekeepers Need Comparison, Not Isolated Data

Weather, flowering and bee activity need to be viewed together rather than as separate pieces of information.

02 Prediction Should Show Uncertainty, Not a Single Answer

Ecological change is uncertain, so the system should show ranges, trends and possible scenarios instead of one fixed result.

03 The System Should Support Experience, Not Replace It

Bee Cue provides decision support while leaving final judgment to beekeepers and field practitioners.

Design Decisions

Make ecological change visible, comparable and traceable.

The goal is to design a web-based predictive support system that helps beekeeping practice read ecological change without turning prediction into a single command.

Make Ecological Data Visible
Make Prediction Understandable
Make Decision Support Non-instructive
System Logic

From Data to Decision Support

Bee Cue combines rule-based ecological models with a machine learning correction layer. Rule-based models translate ecological relationships into readable indexes, while machine learning is used to reduce part of the gap between predicted and observed bee activity.

Machine Learning Boundary

Machine learning is used as a correction layer, not as a black-box replacement or automatic decision-making engine.

Website Prototype

Website Prototype

The final outcome is a working web prototype that visualizes bee activity, nectar source changes and mismatch risks through a site-based map interface, data dashboard and scenario support page.

Open Live Website Live prototype on Render
Map Interface

The entry view connects field location, weather, nectar sources and system status in one map-based workspace.

Environmental Layer

This layer presents nectar plants, flowering periods and environmental conditions, showing how resource availability changes across time and space.

Behavior Layer

Bee activity is translated into heatmaps, sensor records and activity curves, making behavioral changes visible and comparable.

Strategy Page

Strategy cards organize possible responses based on the current ecological state, allowing users to compare references before taking action.

Strategy Detail Page

Each strategy expands into detailed guidance, including conditions, operation steps, risk notes and an entry to AI-assisted conversation.

Reflection
What I Learned

This project helped me explore how complex ecological data can be translated into an interactive system that is readable, traceable and useful for practical judgment.

Design Boundary

Bee Cue does not replace beekeepers' experience or provide a single correct answer. It supports users by making ecological changes more visible and comparable.

Next Step

Future development could include more long-term field data, more beekeeping sites, improved mobile experience and stronger user feedback mechanisms.
